Rationale

Design Decisions & Tradeoffs

Every architectural choice has a reason — here's the "why" behind the pipeline.

Why Brier Score Over Accuracy?

For probability outputs, accuracy is meaningless — a model predicting 51% for every game achieves ~50% accuracy but is useless. Brier score measures calibration: are 70% predictions actually winning 70% of the time? It penalizes overconfidence quadratically, rewarding honest uncertainty estimates over false precision.

EVALUATION
Why Time-Based CV Instead of Random K-Fold?

Random k-fold leaks future games into training folds, inflating AUC by 3-5%. A June 2025 game in training lets the model see post-trade-deadline rosters when predicting April games. Time-based splits respect temporal ordering — each fold only trains on games before the validation window, mimicking real deployment.

DATA LEAKAGE PREVENTION
Why Retrain at the Trade Deadline?

Concept drift: teams fundamentally change rosters mid-season. The July 30 trade deadline reshuffles talent across the league. We trigger retraining when Brier > 0.24 for 7 consecutive days, PSI > 0.1 on key features, or trade deadline + 48 hours — whichever fires first.

CONCEPT DRIFT
Why Canary Deploy Instead of A/B?

A/B testing splits live traffic, which is risky when predictions affect downstream decisions. Shadow mode runs the challenger model against the same games as the champion, comparing outputs without exposure. Auto-promote after 50+ games if the challenger's Brier score beats the champion.

SAFE DEPLOYMENT
Why a Feature Store?

Train/serve skew is the #1 cause of model degradation in production ML. Feast stores transformation logic once and executes it identically for both training and serving paths. Rolling wOBA uses the same 30-day window logic whether computing historical features or real-time predictions — no subtle divergence.

Sabermetric Feature Engineering (Python) 
import pandas as pd
import numpy as np
from feast import FeatureStore

def compute_woba(df: pd.DataFrame, window: int = 30) -> pd.Series:
    """Weighted on-base average using linear weights (2024 coefficients)."""
    weights = {'bb': 0.69, 'hbp': 0.72, 'single': 0.89,
               'double': 1.27, 'triple': 1.62, 'hr': 2.10}
    numer = sum(weights[e] * df[e].rolling(window).sum() for e in weights)
    return numer / df['pa'].rolling(window).sum()

def compute_fip(df: pd.DataFrame) -> pd.Series:
    """Fielding-Independent Pitching — isolates pitcher from defense."""
    cfip = 3.17  # League-specific constant (2024 AL)
    return (13 * df['hr_allowed'] + 3 * (df['bb_allowed'] + df['hbp_allowed'])
            - 2 * df['strikeouts']) / df['ip'] + cfip

def compute_babip(df: pd.DataFrame) -> pd.Series:
    """Batting avg on balls in play — luck indicator (~.300 baseline)."""
    return (df['hits'] - df['hr']) / (df['ab'] - df['strikeouts'] - df['hr'] + df['sf'])

def park_adjusted_era(era: pd.Series, park_factor: pd.Series) -> pd.Series:
    """Normalize ERA by park effects — Coors inflates, Petco deflates."""
    return era / park_factor

def build_matchup_features(away: pd.DataFrame, home: pd.DataFrame,
                            park: float, rest_days: int) -> dict:
    """Assemble 120-feature vector for a single game prediction."""
    store = FeatureStore(repo_path="/opt/feast/mlb_features")
    features = store.get_online_features(
        feature_refs=["team_batting:rolling_woba_30d",
                      "team_pitching:rolling_fip_14d",
                      "team_pitching:bullpen_fatigue_idx"],
        entity_rows=[{"team_id": away['team_id']},
                     {"team_id": home['team_id']}]
    ).to_dict()
    return {'away_woba': features['rolling_woba_30d'][0],
            'home_woba': features['rolling_woba_30d'][1],
            'away_fip': compute_fip(away).iloc[-1],
            'home_era_adj': park_adjusted_era(home['era'], park).iloc[-1],
            'rest_delta': rest_days, 'park_factor': park}
XGBoost Training with Time-Based CV (Python) 
import xgboost as xgb
import optuna
import mlflow
from sklearn.model_selection import TimeSeriesSplit
from sklearn.metrics import brier_score_loss, roc_auc_score

def objective(trial: optuna.Trial, X: pd.DataFrame, y: pd.Series) -> float:
    """Optuna objective — minimize Brier score with time-based CV."""
    params = {
        'max_depth': trial.suggest_int('max_depth', 4, 8),
        'learning_rate': trial.suggest_float('lr', 0.01, 0.1, log=True),
        'n_estimators': trial.suggest_int('n_est', 400, 1200),
        'subsample': trial.suggest_float('sub', 0.6, 0.9),
        'colsample_bytree': trial.suggest_float('col', 0.5, 0.8),
        'reg_alpha': trial.suggest_float('alpha', 1e-3, 1.0, log=True),
        'reg_lambda': trial.suggest_float('lambda', 1e-3, 10.0, log=True),
        'objective': 'binary:logistic',
        'eval_metric': 'auc',
        'tree_method': 'gpu_hist',
    }
    tscv = TimeSeriesSplit(n_splits=5)
    brier_scores = []
    for train_idx, val_idx in tscv.split(X):
        model = xgb.XGBClassifier(**params)
        model.fit(X.iloc[train_idx], y.iloc[train_idx],
                  eval_set=[(X.iloc[val_idx], y.iloc[val_idx])],
                  verbose=False)
        proba = model.predict_proba(X.iloc[val_idx])[:, 1]
        brier_scores.append(brier_score_loss(y.iloc[val_idx], proba))
    return np.mean(brier_scores)

def train_champion(X: pd.DataFrame, y: pd.Series) -> xgb.XGBClassifier:
    """Full training loop: Optuna search → MLflow logging → best model."""
    study = optuna.create_study(direction='minimize',
                                sampler=optuna.samplers.TPESampler(seed=42))
    study.optimize(lambda t: objective(t, X, y), n_trials=200)
    best = study.best_params
    with mlflow.start_run(run_name=f"xgb_v{mlflow.active_run().info.run_id[:6]}"):
        mlflow.log_params(best)
        model = xgb.XGBClassifier(**best, objective='binary:logistic')
        model.fit(X, y)
        mlflow.xgboost.log_model(model, "model")
        mlflow.log_metric('best_brier', study.best_value)
    return model
Drift Detection with Evidently (Python) 
from evidently.report import Report
from evidently.metric_preset import DataDriftPreset
from evidently.metrics import ColumnDriftMetric
import datetime

TRADE_DEADLINE = datetime.date(2026, 7, 30)
BRIER_THRESHOLD = 0.24
PSI_THRESHOLD = 0.10
BRIER_WINDOW_DAYS = 7

def check_drift(reference: pd.DataFrame, current: pd.DataFrame,
                rolling_brier: list[float]) -> dict:
    """Evaluate feature drift (PSI), concept drift (Brier), and calendar triggers."""
    report = Report(metrics=[
        DataDriftPreset(),
        ColumnDriftMetric(column_name='rolling_woba_30d', stattest='psi'),
        ColumnDriftMetric(column_name='rolling_fip_14d', stattest='psi'),
        ColumnDriftMetric(column_name='bullpen_fatigue_idx', stattest='psi'),
    ])
    report.run(reference_data=reference, current_data=current)
    results = report.as_dict()

    # Extract PSI values for key features
    psi_vals = {m['result']['column_name']: m['result']['drift_score']
                for m in results['metrics'] if 'drift_score' in m.get('result', {})}
    max_psi = max(psi_vals.values(), default=0)

    # Check all three retrain triggers
    brier_trigger = (len(rolling_brier) >= BRIER_WINDOW_DAYS and
                     all(b > BRIER_THRESHOLD for b in rolling_brier[-BRIER_WINDOW_DAYS:]))
    psi_trigger = max_psi > PSI_THRESHOLD
    deadline_trigger = (
        datetime.date.today() - TRADE_DEADLINE
    ).days in range(0, 3)  # Within 48h of deadline

    should_retrain = brier_trigger or psi_trigger or deadline_trigger
    return {'retrain': should_retrain, 'max_psi': max_psi,
            'trigger': ('brier' if brier_trigger else
                        'psi' if psi_trigger else
                        'deadline' if deadline_trigger else 'none'),
            'psi_breakdown': psi_vals}

Innovation Spotlight

Log5 Ensemble with Vegas Calibration — Combining Bill James' Log5 model (a Bayesian approach to head-to-head win probability from team records) with XGBoost sabermetric outputs, then calibrating the ensemble against Vegas implied probabilities derived from moneyline odds. The ensemble identifies games where the model's edge exceeds the vig — positive expected value opportunities that pure ML or pure market approaches miss individually.

Uncertainty Quantification via Conformal Prediction — Instead of bare point estimates ("64.3% win probability"), the pipeline outputs prediction intervals (e.g., 64.3% ± 8.2%) using conformal prediction. This provides distribution-free coverage guarantees: the true probability falls within the interval 90% of the time, letting consumers of the prediction gauge whether the model is confident or hedging.
