Multi-Format Scoring Engine

Dynamic PPR / Half-PPR / Standard with position-specific stat weighting

QB: passYds × 0.04 + passTD × 4 + INT × (−2) + rushYds × 0.1 + rushTD × 6
RB: rushYds × 0.1 + rushTD × 6 + recYds × 0.1 + recTD × 6 + receptions × format
WR/TE: recYds × 0.1 + recTD × 6 + receptions × format + rushYds × 0.1
K: FG_made × 3 + XP_made × 1
DST: sacks × 1 + INT × 2 + FR × 2 + TD × 6 + PA bracket
Format multiplier: PPR = 1.0 | Half = 0.5 | Standard = 0.0 per reception
function fantasyPts(player, fmt) { const rBonus = fmt==='ppr' ? 1 : fmt==='half' ? 0.5 : 0; if (player.pos === 'QB') return passYd*.04 + passTD*4 + INT*-2 + rushYd*.1 + rushTD*6; return rec*rBonus + recYd*.1 + recTD*6 + rushYd*.1 + rushTD*6; }

VOR Tiering & Replacement Value

Value Over Replacement Player (VORP) — the core of premium draft strategy

VOR = Player's projected points − replacement-level baseline at position
Replacement thresholds: QB13, RB25, WR31, TE13, K10, DST10 (12-team calibrated)
VOR normalizes cross-position value: a TE1 vs. RB15 vs. WR20
Tier breaks at natural cluster gaps in VOR distribution
Rankings dynamically shift when scoring format changes (high-rec RBs rise in PPR)
// Replacement level = pts of Nth-ranked player at position const repLvl = { QB: 13, RB: 25, WR: 31, TE: 13, K: 10, DST: 10 }; player.vor = player.pts - replacementPts[player.pos]; // Positive VOR = above replacement = draftable

Snake Draft Engine

12-team, 19-round snake draft — configurable position and scoring

Snake order: 1→12, 12→1, 1→12 (even rounds reverse)
User selects draft position (1st through 12th overall)
Roster slots: 3 QB, 5 RB, 5 WR, 2 TE, 2 K, 2 DST = 19 rounds
Real-time ADP comparison: value picks (green) vs. reaches (red)
AI applies ±8% randomness to prevent identical drafts across runs

Technical Interview Talking Point

"I built a production-grade fantasy football draft simulator that demonstrates full-stack data engineering: it ingests 220+ players across 6 positions (QB, RB, WR, TE, K, DST) with projected 2026 stats via SportsData.io API integration, runs a multi-format scoring engine (PPR, Half-PPR, Standard) with dynamic VOR tiering, then executes a 12-team, 19-round snake draft where 11 AI bots each run a distinct strategy archetype — Zero RB, Hero RB, TE Premium, Robust RB, Contrarian, Stars & Scrubs, Late QB, Anchor, and Balanced BPA. The AI pick function computes strategy_weight × VOR × need_factor × scarcity_bonus ± noise per candidate. Post-draft analytics include projected points, ADP value score, bye-week conflict analysis, positional strength bars, and A+→F grading across all teams. Roster construction mirrors real fantasy leagues: 3 QB, 5 RB, 5 WR, 2 TE, 2 K, 2 DST."

Key Engineering Decisions
  • Dynamic VOR recalculation — Switching scoring format triggers a full re-sort of the 220-player pool, because WR/RB rankings shift meaningfully between PPR and Standard. High-reception RBs like Achane and Gibbs gain ~40 fantasy points in PPR vs. Standard.
  • Strategy diversity via weight vectors — Rather than hardcoding pick behavior, each AI archetype applies a 6-element weight vector [QB, RB, WR, TE, K, DST] that multiplies each available player's VOR score, creating naturally divergent rosters.
  • Positional scarcity as a 2nd-order signal — The AI counts how many players at each position remain in the top-N available. When a position falls below a threshold, a scarcity multiplier kicks in (1.25×-1.5×) creating realistic "position run" behavior.
  • ADP-aware value detection — Each pick is compared to ADP. Picks made significantly above ADP are flagged as value picks; picks made below ADP are flagged as reaches. This mirrors ESPN's draft tracker.
