NFL Data API Integration

Real-time player stats from ESPN's public endpoints

ESPN Athletes API — 300+ skill position players
Passing, rushing, receiving stats per season
Player metadata: team, age, experience, position
Headshot images via ESPN CDN
Pagination & rate-limit handling built-in

Composite Scoring Engine

Multi-factor ranking with positional normalization

Weighted stat composites per position group
QB: pass yds (40%) + TDs (35%) + rating (25%)
RB: rush yds (45%) + TDs (30%) + rec yds (25%)
WR/TE: rec yds (50%) + TDs (30%) + catches (20%)
Min-max normalization → 0-100 draft score

Snake Draft Simulation

You pick 1st — 10 AI bots pick via snake order

15 rounds × 10 teams = 150 total picks
Snake order: 1→10, 10→1, 1→10, …
User picks first every odd round
AI drafters use independent strategies
Real-time draft board with animated picks

ML Draft Logic

Value Over Replacement + positional scarcity

VOR = player score − replacement-level baseline
Positional scarcity: remaining pool depth analysis
Need-based weighting: team roster composition
BPA vs. need hybrid optimization per round
Each bot has unique strategy bias (0.3-0.7)

Technical Talking Point

"I built an NFL draft simulator that ingests 300+ players from ESPN's public API, computes composite scores with position-specific weighting (pass yds, rush yds, rec yds, TDs, passer rating), then runs a 10-team snake draft where each AI bot uses a different BPA-vs-need strategy weighted by Value Over Replacement and real-time positional scarcity analysis. The user drafts 1st overall and the engine evaluates all teams with letter grades based on total VOR, positional coverage, and draft efficiency."
