Processing Phases

Each phase uses specialized AI models and rule engines for maximum accuracy.

Document Intake

Multi-channel ingestion with deduplication

Email attachment extraction (IMAP)
REST API upload (multi-part, batch)
SFTP / watched folder polling
Dedup via perceptual hashing
PDF, TIFF, PNG, JPEG, HEIC supported

Classification

ML-based document type identification

LayoutLM / BERT document classifier
12 doc types (invoice, PO, receipt, etc.)
Confidence scoring with threshold routing
Low-confidence → human review queue
98.5% classification accuracy

OCR & Text Extraction

Multi-engine OCR with layout analysis

Azure Document Intelligence (Form Recognizer)
Tesseract fallback for edge cases
Table detection & reconstruction
Handwriting recognition (ICR)
Pre-processing: deskew, denoise, binarize

Entity Extraction

Structured data from unstructured documents

Named Entity Recognition (vendor, amount, date)
Key-value pair extraction (invoice fields)
Line item table parsing (qty, unit, total)
GPT-4o fallback for complex layouts
Custom models per vendor template

Validation & Rules

Business rule engine with cross-referencing

Field-level validation (date, amount format)
PO matching (3-way: PO, receipt, invoice)
Duplicate detection (invoice # + vendor)
Tolerance thresholds (±$0.50 variance)
Configurable rules per document type

Human-in-the-Loop

Exception handling & active learning

Review queue for low-confidence extractions
Side-by-side: original doc + extracted data
One-click correction UI
Corrections feed back to model training
85% straight-through (no human needed)
