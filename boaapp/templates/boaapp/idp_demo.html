Architecture

Design Decisions & Tradeoffs

Key engineering decisions backed by production data and cost analysis.

Automation Threshold

Why 95% Confidence for Straight-Through Processing?

Below 95%, extraction errors compound: a misread amount cascades into wrong GL coding, wrong payment, and reconciliation failures. At 95%+ confidence, the error rate drops to 0.3%, which is cheaper than the labor cost of human review ($4.50/invoice). Below 95%, routing to HITL costs $4.50 but prevents average $847 in payment errors.

Reliability Pattern

Why Multi-Engine OCR (Azure + Tesseract Fallback)?

Azure Document Intelligence achieves 99.2% accuracy on clean prints but drops to 87% on handwritten annotations, faded receipts, or skewed scans. Tesseract re-runs failed pages with deskew+binarize preprocessing, recovering 40% of Azure failures. Dual-engine cost is only $0.003/page more but increases throughput by 12%.

Continuous Improvement

Why Active Learning Instead of Static Models?

HITL corrections create labeled training data automatically. Every time a human fixes an extraction error, that correction becomes a training example. Over 6 months, the model improves from 88% to 96% straight-through rate — purely from production corrections, with zero manual annotation effort.

Fraud Prevention

Why 3-Way PO Matching?

Cross-referencing Purchase Order + Goods Receipt + Invoice prevents 3 fraud vectors: (a) duplicate invoices for the same PO, (b) invoices for goods never received, (c) inflated amounts beyond PO authorization. This catches 94% of AP fraud before payment, with 2.1% false positive rate.

AI Classification

Why Claude for GL Account Classification?

Rule-based GL mapping requires maintaining 500+ vendor→account rules that break when vendors change names or descriptions. Claude infers GL accounts from context: "Office Depot — printer paper, toner, desk organizer" → 6200 (Office Supplies) with 97.3% accuracy, vs. 82% for rule-based. Zero maintenance when new vendors appear.

Azure Document Intelligence + Tesseract Pipeline (Python) 
import cv2
import numpy as np
from azure.ai.formrecognizer import DocumentAnalysisClient
from azure.identity import DefaultAzureCredential
import pytesseract
from PIL import Image

class DualEngineOCR:
    """Multi-engine OCR: Azure primary, Tesseract fallback for low-confidence pages."""

    def __init__(self, endpoint: str, confidence_threshold: float = 0.85):
        credential = DefaultAzureCredential()
        self.client = DocumentAnalysisClient(endpoint, credential)
        self.threshold = confidence_threshold

    def preprocess(self, image_path: str) -> np.ndarray:
        """Deskew and binarize for Tesseract fallback."""
        img = cv2.imread(image_path, cv2.IMREAD_GRAYSCALE)
        # Deskew via Hough line detection
        coords = np.column_stack(np.where(img < 128))
        angle = cv2.minAreaRect(coords)[-1]
        if angle < -45:
            angle = -(90 + angle)
        else:
            angle = -angle
        h, w = img.shape
        M = cv2.getRotationMatrix2D((w // 2, h // 2), angle, 1.0)
        img = cv2.warpAffine(img, M, (w, h), flags=cv2.INTER_CUBIC,
                             borderMode=cv2.BORDER_REPLICATE)
        # Adaptive binarization
        img = cv2.adaptiveThreshold(img, 255, cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
                                    cv2.THRESH_BINARY, 31, 11)
        return img

    async def extract(self, doc_path: str) -> dict:
        """Run Azure first; fall back to Tesseract on low-confidence pages."""
        with open(doc_path, "rb") as f:
            poller = await self.client.begin_analyze_document(
                "prebuilt-invoice", document=f
            )
        result = await poller.result()

        pages, fallback_pages = [], []
        for page in result.pages:
            avg_conf = np.mean([w.confidence for w in page.words]) if page.words else 0
            if avg_conf >= self.threshold:
                pages.append({"engine": "azure", "confidence": avg_conf,
                              "words": [w.content for w in page.words]})
            else:
                fallback_pages.append(page.page_number)

        # Tesseract fallback for low-confidence pages
        for page_num in fallback_pages:
            preprocessed = self.preprocess(f"{doc_path}_page{page_num}.png")
            text = pytesseract.image_to_data(
                Image.fromarray(preprocessed), output_type=pytesseract.Output.DICT
            )
            conf_vals = [int(c) for c in text["conf"] if int(c) > 0]
            pages.append({"engine": "tesseract", "page": page_num,
                          "confidence": np.mean(conf_vals) / 100 if conf_vals else 0,
                          "words": [w for w, c in zip(text["text"], text["conf"])
                                    if int(c) > 40 and w.strip()]})

        return {"pages": pages, "fallback_count": len(fallback_pages),
                "avg_confidence": np.mean([p["confidence"] for p in pages])}
Active Learning Training Loop (Python) 
import torch
from torch.utils.data import DataLoader, WeightedRandomSampler
from transformers import LayoutLMv3ForTokenClassification, LayoutLMv3Processor
from datetime import datetime, timedelta
import json

class ActiveLearningPipeline:
    """Ingest HITL corrections and fine-tune extraction model."""

    def __init__(self, model_name: str, corrections_db):
        self.processor = LayoutLMv3Processor.from_pretrained(model_name)
        self.model = LayoutLMv3ForTokenClassification.from_pretrained(model_name)
        self.corrections_db = corrections_db

    def ingest_corrections(self, since: timedelta = timedelta(days=7)) -> list:
        """Extract correction pairs from HITL review queue."""
        cutoff = datetime.utcnow() - since
        corrections = self.corrections_db.find({
            "reviewed_at": {"$gte": cutoff},
            "status": "corrected"
        })
        training_pairs = []
        for c in corrections:
            training_pairs.append({
                "document_id": c["document_id"],
                "original_extraction": c["model_output"],
                "corrected_extraction": c["human_correction"],
                "field": c["field_name"],
                "confidence_delta": c["original_confidence"]
            })
        return training_pairs

    def confidence_weighted_loss(self, logits, labels, confidences):
        """Weight loss inversely by original confidence — learn more from
        low-confidence errors that humans corrected."""
        base_loss = torch.nn.functional.cross_entropy(
            logits.view(-1, logits.size(-1)), labels.view(-1), reduction="none"
        )
        weights = 1.0 - confidences.view(-1).clamp(0.5, 0.99)
        weights = weights / weights.sum() * len(weights)
        return (base_loss * weights).mean()

    def retrain(self, training_pairs: list, epochs: int = 3,
                lr: float = 2e-5) -> dict:
        """Fine-tune model on HITL corrections with confidence weighting."""
        dataset = self._build_dataset(training_pairs)
        # Oversample rare correction types
        weights = [1.0 / max(p["confidence_delta"], 0.01) for p in training_pairs]
        sampler = WeightedRandomSampler(weights, len(weights))
        loader = DataLoader(dataset, batch_size=8, sampler=sampler)

        optimizer = torch.optim.AdamW(self.model.parameters(), lr=lr)
        self.model.train()
        metrics = {"epoch_losses": []}

        for epoch in range(epochs):
            total_loss = 0
            for batch in loader:
                logits = self.model(**batch["inputs"]).logits
                loss = self.confidence_weighted_loss(
                    logits, batch["labels"], batch["confidences"]
                )
                loss.backward()
                optimizer.step()
                optimizer.zero_grad()
                total_loss += loss.item()
            metrics["epoch_losses"].append(total_loss / len(loader))

        return {"corrections_used": len(training_pairs),
                "final_loss": metrics["epoch_losses"][-1],
                "metrics": metrics}
3-Way PO Match Validation (Python) 
import hashlib
from dataclasses import dataclass
from typing import Optional
import imagehash
from PIL import Image

@dataclass
class MatchResult:
    status: str          # "matched" | "variance" | "no_po" | "no_receipt" | "duplicate"
    variance_pct: float
    details: dict

class ThreeWayMatcher:
    """Cross-reference PO + Goods Receipt + Invoice for fraud prevention."""

    AMOUNT_TOLERANCE = 0.02   # ±2% for currency rounding
    PHASH_THRESHOLD = 6       # Hamming distance for visual duplicate

    def __init__(self, po_store, receipt_store, invoice_store):
        self.po_store = po_store
        self.receipt_store = receipt_store
        self.invoice_store = invoice_store

    def detect_visual_duplicate(self, invoice_path: str) -> Optional[str]:
        """Perceptual hash to catch re-submitted PDFs with different filenames."""
        current_hash = imagehash.phash(Image.open(invoice_path))
        existing = self.invoice_store.find({"status": "processed"})
        for inv in existing:
            stored = imagehash.hex_to_hash(inv["phash"])
            if current_hash - stored <= self.PHASH_THRESHOLD:
                return inv["invoice_number"]
        return None

    def match(self, invoice: dict) -> MatchResult:
        """Full 3-way match: PO → Goods Receipt → Invoice."""
        # 1. Check for visual duplicates via pHash
        if invoice.get("file_path"):
            dup = self.detect_visual_duplicate(invoice["file_path"])
            if dup:
                return MatchResult(
                    status="duplicate", variance_pct=0,
                    details={"duplicate_of": dup, "method": "perceptual_hash"}
                )

        # 2. Check for invoice number duplicates
        existing = self.invoice_store.find_one({
            "invoice_number": invoice["invoice_number"],
            "vendor_id": invoice["vendor_id"],
            "status": {"$ne": "rejected"}
        })
        if existing:
            return MatchResult(
                status="duplicate", variance_pct=0,
                details={"duplicate_of": existing["invoice_number"],
                         "method": "invoice_number+vendor"}
            )

        # 3. PO lookup
        po = self.po_store.find_one({"po_number": invoice["po_number"]})
        if not po:
            return MatchResult(
                status="no_po", variance_pct=0,
                details={"po_number": invoice["po_number"]}
            )

        # 4. Goods receipt confirmation
        receipt = self.receipt_store.find_one({
            "po_number": invoice["po_number"],
            "status": "received"
        })
        if not receipt:
            return MatchResult(
                status="no_receipt", variance_pct=0,
                details={"po_number": invoice["po_number"],
                         "reason": "goods not yet received"}
            )

        # 5. Amount matching with tolerance
        po_amount = po["total_amount"]
        inv_amount = invoice["total_amount"]
        variance = abs(inv_amount - po_amount) / po_amount if po_amount else 0

        if variance > self.AMOUNT_TOLERANCE:
            return MatchResult(
                status="variance", variance_pct=round(variance * 100, 2),
                details={"po_amount": po_amount, "invoice_amount": inv_amount,
                         "threshold": f"{self.AMOUNT_TOLERANCE*100}%"}
            )

        return MatchResult(
            status="matched", variance_pct=round(variance * 100, 2),
            details={"po": po["po_number"], "receipt": receipt["receipt_id"],
                     "invoice": invoice["invoice_number"],
                     "amount_verified": inv_amount}
        )

Innovation Spotlight
Perceptual Hashing for Duplicate Detection

Using pHash to detect invoices that are visually identical (same PDF re-submitted) even when filenames differ, catching 7.2% of duplicate payment attempts. Hamming distance ≤6 flags duplicates; combined with invoice-number dedup, this prevents $142K/year in overpayments.

Confidence Calibration via Platt Scaling

Raw OCR confidence scores are poorly calibrated — a model saying "95% confident" may only be right 88% of the time. Platt scaling (logistic regression on a holdout set) transforms scores so that "95% confident" genuinely means 95% accuracy, enabling reliable STP thresholds and reducing unnecessary HITL routing by 23%.
